# 三目並べ（Tic-Tac-Toe）

2人のユーザがネットワーク越しに対戦できる三目並べです。
HTTP サーバ・コントローラ・GUI クライアントの3層構成になっています。
すべて Python 標準ライブラリのみで動作し、追加インストールは不要です。

## ファイル構成

| ファイル | 役割 |
| --- | --- |
| `controller.py` | ゲームのルール・状態管理（HTTP に非依存） |
| `server.py` | HTTP サーバ。リクエスト受付・認証・JSON 変換 |
| `client.py` | tkinter による GUI クライアント |

## 必要環境

- Python 3.7 以上
- `tkinter`（通常は Python に同梱。Linux で未導入の場合は別途インストール）
  - Debian / Ubuntu: `sudo apt install python3-tk`

## 使い方

### 1. サーバを起動する

環境変数で次の設定ができます。

- `ACCESS_TOKEN` … アクセストークン。設定するとリクエストにトークン一致が必須。
- `PORT` … 待ち受けポート番号（未設定なら `8000`）。

Linux / macOS:

​```bash
ACCESS_TOKEN=secret123 PORT=9000 python server.py
​```

Windows (PowerShell):

​```powershell
$env:ACCESS_TOKEN="secret123"; $env:PORT="9000"; python server.py
​```


トークンを設定せずに起動すると、認証なし（誰でも接続可）モードで動作します。

```bash
python server.py
```

起動すると待ち受けアドレスと認証モードが表示されます。

```
Server running at http://127.0.0.1:8000  [認証あり]
```

### 2. クライアントを2つ起動する

別ウィンドウとして2つ起動します（同一マシンでも別マシンでも可）。

```bash
python client.py
```

### 3. 接続して対戦する

1. 設定画面で **接続先 URL**（既定 `http://127.0.0.1:8000`）を入力します。
2. サーバでトークンを設定した場合は **Access Token** に同じ値を入力します。
3. **「接続して参加」** を押すとマークが割り当てられます（先着が `X`、次が `O`）。
4. 2人揃うまでは盤面が無効化され「対戦相手の参加を待っています...」と表示されます。
5. 2人揃うと自動的に対局が始まります。自分の手番のときだけマスをクリックできます。
6. 勝敗が決まると結果が表示され、盤面が無効化されます。

## 各プログラムの説明

### controller.py

ゲームの中核ロジックを担う `Controller` クラスを定義します。
HTTP には一切依存しないため、単体テストが容易です。

- 盤面（9マス）、手番、勝者、参加プレイヤーを状態として保持します。
- `join()` … プレイヤーを登録し、マーク（`X` / `O`）とトークンを返します。最大2人。
- `move(token, pos)` … 指定プレイヤーの着手を処理します。2人揃う前・手番違い・
  ゲーム終了後・使用済みマスなどの不正な手はすべて拒否します。
- `reset()` … 盤面を初期化します。
- 勝敗判定は縦・横・斜めの8ラインをチェックし、全マス埋まりで引き分けとします。
- 状態は `board` / `turn` / `winner` / `ready`（2人揃ったか）/ `players`（接続人数）を
  含むスナップショットとして返します。

### server.py

`http.server` をベースにした HTTP サーバです。
`ThreadingHTTPServer` で複数クライアントを同時に処理し、状態更新は
`threading.Lock` で直列化して競合を防ぎます。

提供するエンドポイント:

| メソッド | パス | 説明 |
| --- | --- | --- |
| POST | `/join` | ゲームに参加し、マークとトークンを取得 |
| POST | `/move` | 着手する（JSON: `{"player": token, "pos": 0-8}`） |
| GET | `/state` | 現在の盤面・手番・勝者・接続状況を取得 |
| POST | `/reset` | 盤面を初期化 |

認証:

- 環境変数 `ACCESS_TOKEN` が設定されている場合のみ認証を要求します。
- クライアントは `Authorization: Bearer <token>` または `X-Access-Token` ヘッダで
  トークンを送ります。
- トークンの比較は `hmac.compare_digest` による定数時間比較で行い、
  タイミング攻撃を防ぎます。
- 不一致の場合は `401 unauthorized` を返してアクセスを拒否します。

### client.py

tkinter による GUI クライアントです。通信処理（`ApiClient`）と
画面処理（`TicTacToeGUI`）に分かれています。

- **設定画面**: 接続先 URL と Access Token を入力します。
- **ゲーム画面**: 3×3 のボタンで盤面を表示し、状態をステータス欄に表示します。
- 状態は1秒ごとにサーバへポーリングし、自動更新します。
- ネットワーク処理は別スレッドで実行するため、通信中も UI が固まりません。
- 2人揃うまでは全マスを無効化し、待機メッセージを表示します。
- 対局中は自分の手番かつ空いているマスのみクリック可能です。

## 通信フローの概要

```
[クライアントX] --join--> [server] --> [controller] (X を割当)
[クライアントO] --join--> [server] --> [controller] (O を割当 → ready)
[クライアント]  --state--> [server] --> [controller] (盤面取得、毎秒ポーリング)
[クライアント]  --move---> [server] --> [controller] (着手・ルール検証)
```

## 補足・拡張のヒント

- 本実装は学習・ローカル利用向けの簡易版です。トークンはプレイヤー名固定
  （`player-X` / `player-O`）になっています。
- より堅牢にするには、ランダムなプレイヤートークンの個別発行、
  ロングポーリングや WebSocket による即時更新、HTTPS 化、
  複数ゲームのルーム管理などを追加できます。
